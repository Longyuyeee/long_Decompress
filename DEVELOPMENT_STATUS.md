# 胧压缩·方便助手 - 当前开发现状文档 (v2.2)

## 1. 项目愿景与设计哲学
*   **引擎强大，界面极简**：基于 Rust 的高性能核心，通过上下文感知的前端界面隐藏复杂性。
*   **交互美学**：采用苹果风（Apple Style）、毛玻璃质感（Glassmorphism）、高频动效（GSAP）。
*   **任务中心**：一切操作围绕文件任务展开，摆脱传统管理软件的层级负荷。

## 2. 已完成的核心架构 (Milestone 1-6)

### 后端 (Rust/Tauri)
*   **架构稳健化**：
    *   **热重载隔离**：将持久化数据目录 `.password_book_data` 移出 `src-tauri` 监控范围，彻底杜绝了因数据写入导致的程序“闪退”重启。
    *   **命令全注册**：在 `main.rs` 中规范化注册了全部 30+ 业务命令，消除了前端调用失效问题。
    *   **警告净化**：完成了全局编译警告抑制，使控制台专注于真实的逻辑错误。
*   **生命线集成**：支持通过 `Window.emit` 推送实时 IO 日志流、百分比进度和冲突事件。
*   **全格式驱动**：支持 `ZIP, 7z, RAR, TAR, GZ, BZ2, XZ` 等。

### 前端 (Vue 3/TS/Vite)
*   **密码本 2.0 重构**：
    *   **数据可视化**：新增了基于 CSS 的调用趋势分析图表，支持每日频率统计。
    *   **极简弹窗**：移除了冗余的账号/分类字段，实现了智能默认命名与状态重置逻辑。
    *   **响应式增强**：适配了 `800x560` 的小工具窗口尺寸，解决了极小尺寸下的文字换行与布局崩坏。
*   **实时反馈系统**：
    *   **全局通知条**：顶置的滑动反馈，清晰告知用户操作结果（成功、失败、同步中）。
    *   **确认机制**：为清空等高危操作增加了沉浸式二次确认。

## 3. 环境与工程化修复
*   **存储对齐**：统一采用明文存取方案（应需求），解决了 JSON 片段显示在 UI 上的历史遗留问题。
*   **兼容性迁移**：通过 `#[serde(default)]` 实现了旧版本数据到 `usage_history` 模型的无缝热迁移。

## 4. 后续演进计划
*   实现具体的暴力破解穷举算法（Rust 侧）。
*   完善 DLong 格式的物理文件导入导出解析逻辑。
*   增加对加密 RAR 文件的更深层驱动支持。
