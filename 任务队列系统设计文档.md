# 胧压缩·方便助手 - 任务队列系统设计文档

## 1. 概述
任务队列系统负责管理压缩和解压任务的排队、执行、状态跟踪和并发控制。系统需要支持批量处理、优先级调度、进度跟踪和错误恢复。

## 2. 设计目标
1. **并发控制**: 支持多任务并行执行，限制最大并发数
2. **优先级调度**: 支持任务优先级（高、中、低）
3. **状态管理**: 实时跟踪任务状态和进度
4. **错误处理**: 支持任务失败重试和错误恢复
5. **持久化**: 任务状态持久化存储，支持应用重启恢复
6. **通知机制**: 任务状态变化通知前端

## 3. 系统架构

### 3.1 核心组件
```
├── task_queue/
```

```
前端请求 → Tauri命令 → 任务管理器 → 任务队列 → 任务调度器 → 任务执行器
                                 ↓
                         任务持久化存储
                                 ↓
                         任务状态通知器 → 前端更新
```

## 4. 详细设计

### 4.1 任务管理器 (TaskManager)
**职责**: 系统主入口，协调所有组件
**功能**:
- 初始化队列系统
- 接收任务添加请求
- 管理任务生命周期
- 提供任务查询接口

### 4.2 任务队列 (TaskQueue)
**职责**: 存储和管理待执行任务
**特性**:
- 优先级队列（高、中、低）
- 线程安全访问
- 任务去重检查
- 队列容量限制

### 4.3 任务调度器 (TaskScheduler)
**职责**: 决定任务执行顺序和时间
**策略**:
- 基于优先级的调度
- 并发任务数限制
- 资源感知调度（CPU/内存）
- 任务依赖关系处理

### 4.4 任务执行器 (TaskExecutor)
**职责**: 实际执行压缩/解压任务
**特性**:
- 异步任务执行
- 进度回调支持
- 错误处理和重试
- 资源使用监控

### 4.5 任务通知器 (TaskNotifier)
**职责**: 通知前端任务状态变化
**机制**:
- WebSocket/Tauri事件
- 状态变化实时推送
- 批量更新优化
- 断线重连支持

### 4.6 任务持久化 (TaskPersistence)
**职责**: 持久化存储任务状态
**存储**:
- SQLite数据库
- 任务状态序列化
- 应用重启恢复
- 历史记录查询

## 7. 错误处理和恢复

### 7.1 错误分类
1. **可恢复错误**: 文件锁定、临时IO错误（自动重试）
2. **部分可恢复错误**: 密码错误、格式不支持（用户交互）
3. **不可恢复错误**: 文件损坏、权限不足（任务失败）

### 7.2 重试策略
- **最大重试次数**: 可配置（默认3次）
- **重试间隔**: 指数退避（1s, 2s, 4s）
- **重试条件**: 仅对可恢复错误重试

## 10. 数据库设计

### 10.1 任务表 (queue_tasks)
```sql
CREATE TABLE queue_tasks (
    id TEXT PRIMARY KEY,
    task_type TEXT NOT NULL,           -- 'compress' 或 'extract'
    priority TEXT NOT NULL,            -- 'high', 'medium', 'low'
    status TEXT NOT NULL,              -- 队列状态
    compression_task_json TEXT NOT NULL, -- 序列化的CompressionTask
    created_at TIMESTAMP NOT NULL,
    scheduled_at TIMESTAMP,
    started_at TIMESTAMP,
    completed_at TIMESTAMP,
    retry_count INTEGER DEFAULT 0,
    max_retries INTEGER DEFAULT 3,
    error_message TEXT,
    INDEX idx_status (status),
    INDEX idx_priority (priority),
    INDEX idx_created_at (created_at)
);
```

### 10.2 任务历史表 (task_history)
```sql
CREATE TABLE task_history (
    id TEXT PRIMARY KEY,
    task_id TEXT NOT NULL,
    status TEXT NOT NULL,
    progress REAL,
    error_message TEXT,
    timestamp TIMESTAMP NOT NULL,
    FOREIGN KEY (task_id) REFERENCES queue_tasks(id),
    INDEX idx_task_id (task_id),
    INDEX idx_timestamp (timestamp)
);
```
