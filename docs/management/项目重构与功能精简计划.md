# 胧压缩·方便助手 - 项目重构与功能精简计划 (瘦身计划)

## 1. 现状分析
目前项目处于从 Python 原型向 Rust+Tauri 架构迁移的后期，存在以下主要问题：
- **目录混乱**: 根目录下 Python 代码、原型脚本与 Rust/Vue 主工程混杂。
- **功能冗余**: 关键功能（如密码管理、文件上传）存在多个实现版本（旧版 vs New版）。
- **视觉噪音**: 侧边栏包含大量开发示例（设计系统、组件演示），干扰核心用户路径。
- **逻辑断层**: 前端代码仍保留大量 `localStorage` 模拟逻辑，未完全切换至 Rust 后端 API。

## 2. 重构目标 (MVP+)
打造一个“简洁、高性能、直观”的正式版，核心功能聚焦于：
1. **智能解压**: 支持主流格式，自动关联密码本。
2. **任务管理**: 实时进度反馈，支持并发与重试。
3. **极简密码本**: 安全、可搜索的加密密码存储。

## 3. 具体实施方案

### 3.1 物理清理 (文件去重)
- **Legacy 归档**: 创建 `/legacy` 文件夹，将根目录下的 `long_decompress/`, `prototype/`, `example_usage.py` 等 Python 相关内容移入其中。
- **组件收拢**:
    - 删除 `PasswordManager.vue`，将 `PasswordManagerNew.vue` 重命名为 `PasswordManager.vue`。
    - 移除所有以 `Example` 结尾的组件和视图文件。
    - 清理 `src-tauri` 下的临时测试脚本 (`quick_test.rs` 等)。

### 3.2 UI/UX 降噪 (侧边栏精简)
- **合并入口**: 将“首页”、“解压”、“压缩”合并为统一的“文件处理”主页面。
- **隐藏未完成功能**: 暂时移除“统计图表”、“导入/导出”等占位按钮，待后端功能对齐后再开放。
- **路径简化**:
    - [保留] 文件处理 (首页)
    - [保留] 密码管理
    - [保留] 任务中心
    - [保留] 系统设置
    - [移除] 设计系统、组件示例

### 3.3 技术统一
- **后端强一致性**: 移除前端组件内所有 `localStorage` 模拟代码，所有数据持久化必须通过 Tauri `invoke` 调用 Rust 后端。
- **API 规范化**: 统一采用 `snake_case` 命名规范（与 Rust 保持一致），移除前端的兼容性别名层。
