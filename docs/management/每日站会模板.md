# 胧压缩·方便助手 - 每日站会模板

## 会议基本信息
- **会议名称**: 每日站会
- **时间**: 每天上午9:30
- **时长**: 15分钟
- **地点**: 团队聊天工具/视频会议
- **参与人员**: 全体团队成员
- **主持人**: 项目管理领队 (轮流主持)

## 会议目标
1. 同步昨日工作进展
2. 明确今日工作计划
3. 识别问题和障碍
4. 协调团队协作

## 会议流程 (15分钟)
1. **开场** (1分钟) - 主持人
   - 确认参会人员
   - 提醒会议时间限制
   - 说明会议议程

2. **进展同步** (10分钟) - 每位成员轮流
   - 昨日完成的工作
   - 今日计划的工作
   - 遇到的问题和障碍
   - 需要的帮助和支持

3. **问题讨论** (3分钟) - 全体
   - 快速讨论关键问题
   - 确定解决方案或后续跟进

4. **会议总结** (1分钟) - 主持人
   - 总结关键决策
   - 确认下一步行动
   - 宣布会议结束

## 站会最佳实践

### 会前准备
1. **提前准备**: 每位成员会前准备好汇报内容
2. **更新状态**: 更新任务状态跟踪表
3. **问题整理**: 提前整理需要讨论的问题

### 会中纪律
1. **准时开始**: 严格按时开始，不等待迟到者
2. **聚焦主题**: 只讨论进展、计划、问题
3. **时间控制**: 每人2-3分钟，不深入讨论技术细节
4. **问题记录**: 记录所有问题，会后专门讨论

### 会后跟进
1. **会议记录**: 主持人会后15分钟内发布会议记录
2. **问题跟踪**: 记录的问题进入问题跟踪系统
3. **行动跟进**: 确认的下一步行动需要跟进
4. **状态更新**: 根据会议内容更新项目状态

## 常见问题处理

### 问题: 成员汇报时间过长
- **处理**: 主持人温和提醒，建议会后单独讨论
- **预防**: 会前强调时间限制，提供汇报模板

### 问题: 技术细节讨论过深
- **处理**: 记录问题，安排专门的技术讨论会议
- **预防**: 明确站会只同步状态，不讨论解决方案

### 问题: 问题没有明确负责人
- **处理**: 现场指定负责人，记录在会议记录中
- **预防**: 问题提出时要求同时提出建议的负责人

### 问题: 进展汇报不清晰
- **处理**: 要求按模板汇报，明确任务ID和完成度
- **预防**: 提供汇报模板，培训团队成员
