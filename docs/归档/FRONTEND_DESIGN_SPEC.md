# 胧压缩·方便助手 - 全流程交互设计规范与技术实现白皮书 (v2.1)

## 1. 核心设计哲学
*   **引擎强大，界面极简**：利用后端已实现的复杂逻辑（SQLite, TaskQueue, FileService），前端通过上下文感知隐藏复杂性，仅在需要时展示高级选项。
*   **以任务为中心**：用户不应关心“页面”，而应关心“我正在处理的文件”。
*   **动态美学**：苹果风（Apple Style）、毛玻璃（Glassmorphism）、高频动效、流体交互。

---

## 2. 视觉与交互基础规范
*   **视觉风格**：
    *   **背景**：全局 `Backdrop Blur` (20px+) 配合半透明白色/深色叠加。
    *   **容器**：圆角矩形（16px+），细腻的 1px 边框（Inner Border）模拟玻璃质感。
    *   **阴影**：多层软阴影（Soft Shadows），模拟物理层级深度。
*   **底部全局工具栏 (Global Context Bar)平衡**：
    *   **系统负载计**：实时显示 CPU/内存占用。点击展开“性能调度盘”，支持滑块调节 Rust 后端线程占用率（从“静默”到“全速”）。
    *   **配置层级**：界面底部的全局配置决定默认行为。

---

## 3. 解压工作区 (Extraction Workspace)

### 3.1 智慧表格布局 (Extraction Aero-Table)
解压表格强调信息密度与实时反馈。不使用松散的卡片，而是精密表格。
*   **列定义**：
    1.  **[图标+文件名]**：左侧固定。图标根据后缀显示（ZIP/7z/RAR 等专属图标）。
    2.  **[源文件路径]**：显示文件当前完整路径，点击可跳转。
    3.  **[局部配置图标区]**：
        *   **钥匙图标**：点击为该文件单独指定密码。
        *   **路径图标**：点击为该文件单独指定输出目录。
        *   **清理图标**：设置解压成功后是否自动移动源码至回收站。
        *   *逻辑*：若用户修改了某项，图标高亮（主题色），否则随全局配置。
    4.  **[实时状态与进度]**：右侧宽区。显示进度条，进度条上方滚动显示当前动作（如：`正在尝试密码本 '123456'...`）。
    5.  **[详情操作]**：末尾包含一个“更多”按钮，用于展开详情。

### 3.2 详情展示与冲突处理
*   **详情卡片 (Details Card)**：点击“更多”后，表格行向下平滑展开卡片，展示：
    *   文件元数据：MD5/SHA256 指纹、压缩算法、原始总大小。
    *   底层日志流：实时展示后端 IO 操作记录。
*   **智能路径冲突 (Conflict Resolution)**：
    *   **触发**：当后端检测到目标路径已存在同名文件。
    *   **交互逻辑**：表格行原地变换，将进度条替换为决策按钮。
    *   **批量冲突**：若大量重复，弹出全局对话框。
        *   **对比信息**：清晰显示重复文件的修改时间、文件大小对比。
        *   **决策流**：用户可选择 `[单独处理]` 或 `[一键应用到所有冲突项]`。

### 3.3 三级密码自动化阶梯 (3-Tier Password Pipeline)
1.  **内置保险箱**：调用后端 SQLite 中命中率最高、最近使用的历史密码。
2.  **外部暴力密码本**：读取设置中导入的 `.txt` 字典库（每行一个密码）。
3.  **算法穷举**：按序列生成密码进行最后尝试。

---

## 4. 压缩工作区 (Compression Workspace)

### 4.1 磁吸表格与打组逻辑 (Compression Table & Grouping)
压缩表格强调组织逻辑，与解压表格结构不同。
*   **动态打组 (Magnetic Grouping)**：
    *   多选文件后点击“成组”，文件行产生“磁铁吸附”动效，合并为一个视觉整体。
    *   组具有统一的高级配置，但可随时“散开”进行微调。
*   **虚拟镜像预览 (Virtual Preview)**：
    *   调节压缩等级时，表格中的“预计体积”实时跳动更新。
*   **列定义**：图标+文件名、源路径、大小、预估大小、配置、详情。

---

## 5. 增强特性与细节

### 5.1 任务控制系统
*   **按钮形态变换 (Morphing)**：开始按钮点击后平滑变为暂停/停止。
*   **后台保护**：点击停止后发送 `CancellationToken`，后端物理回滚，确保不残留临时碎片文件。

### 5.2 灵动通知 (Dynamic Notification)
*   主窗口最小化后，图标变为灵动通知球，通过颜色填充比例显示进度。
*   需要密码时，通知球红光闪烁，点击直接在当前位置弹出输入框。

### 5.3 暴力破解引擎配置
*   支持在设置界面导入任意数量的 `.txt` 密码字典。
*   逻辑：先搜保险箱，再搜外部字典，最后穷举。

---

## 6. 技术实现要点 (Vue3 + Rust/Tauri)
*   **状态管理**：Pinia 负责维护任务队列与全局/局部配置的继承逻辑。
*   **通信**：Tauri Command 驱动。利用 `tauri::Window::emit` 实现后端解压日志向前端详情卡片的实时推送。
*   **动画**：使用 `GSAP` 或 `Vue Transition Group` 实现表格行展开、成组及文件“吸入”特效。
