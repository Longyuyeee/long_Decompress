# 胧解压·方便助手 - 技术架构设计文档

## 1. 概述
"胧解压·方便助手"是一个跨平台的文件解压缩工具，支持多种压缩格式、密码管理、批量处理和用户友好的界面。

## 2. 技术栈选择

### 2.1 编程语言：Python 3.9+
**选择理由：**
- 快速开发和迭代
- 丰富的生态系统和库支持
- 优秀的跨平台能力
- 适合工具类应用开发

### 2.2 GUI框架：PyQt5/PySide6
**选择理由：**
- 成熟的跨平台GUI框架
- 丰富的UI组件库
- 良好的性能和稳定性
- 信号/槽机制适合异步任务

### 2.3 压缩库策略：混合方案
**核心库：**
1. **py7zr** - 7z格式支持
2. **patool** - 通用压缩格式支持
3. **python-libarchive-c** - libarchive绑定
4. **rarfile** - RAR格式支持（需要unrar库）

**备用方案：**
- 系统命令调用（7z, unrar, tar, unzip等）
- 格式检测自动选择最佳解压方式

## 3. 系统架构设计

### 3.1 模块分层架构

```
├── core/                    # 核心解压引擎
│   ├── decompressor.py      # 解压器抽象接口
│   ├── format_detector.py   # 格式检测（魔数检测）
│   ├── password_manager.py  # 密码管理和存储
│   └── progress_tracker.py  # 进度跟踪和回调
├── ui/                      # 用户界面层
│   ├── main_window.py       # 主窗口（拖放支持）
│   ├── file_dialog.py       # 文件选择对话框
│   ├── progress_dialog.py   # 进度显示对话框
│   ├── settings_dialog.py   # 配置设置对话框
│   └── task_list_widget.py  # 任务列表显示
├── task/                    # 任务调度系统
│   ├── task_scheduler.py    # 任务调度器
│   ├── task_queue.py        # 优先级任务队列
│   ├── worker_pool.py       # 工作线程池管理
│   └── task_monitor.py      # 任务状态监控
├── formats/                 # 格式支持模块
│   ├── base_handler.py      # 处理器基类
│   ├── zip_handler.py       # ZIP格式处理
│   ├── rar_handler.py       # RAR格式处理
│   ├── tar_handler.py       # TAR/GZ/BZ2格式处理
│   ├── sevenzip_handler.py  # 7z格式处理
│   └── other_handler.py     # 其他格式处理
├── utils/                   # 工具模块
│   ├── file_utils.py        # 文件操作工具
│   ├── config_manager.py    # 配置管理（JSON）
│   ├── logger.py           # 日志系统
│   └── system_info.py      # 系统信息检测
└── plugins/                 # 插件系统（预留）
    └── plugin_manager.py    # 插件管理器
```

### 3.2 关键设计模式
1. **策略模式** - 不同压缩格式使用不同的解压策略
2. **观察者模式** - 进度更新通知UI组件
3. **工厂模式** - 根据文件格式创建对应的解压器
4. **命令模式** - 解压任务封装为可执行命令对象
5. **单例模式** - 配置管理器和日志系统

## 4. 并发模型设计

### 4.1 线程架构
```
主线程 (UI线程)
    │
    ├── 用户交互处理
    ├── UI更新
    └── 事件分发
    │
工作线程池 (3-5个线程)
    │
    ├── 线程1: 解压任务执行
    ├── 线程2: 解压任务执行
    ├── 线程3: 解压任务执行
    └── ...
    │
任务队列 (优先级队列)
    │
    ├── 高优先级: 用户主动任务
    ├── 中优先级: 批量任务
    └── 低优先级: 后台任务
```

### 4.2 线程间通信
- **信号/槽机制**：PyQt5原生支持，线程安全
- **队列通信**：任务队列传递解压任务
- **回调函数**：进度更新回调到UI线程
- **事件系统**：自定义事件传递状态变化

### 4.3 并发控制
- 最大并发任务数：可配置（默认3个）
- 内存限制：大文件单任务处理
- CPU占用控制：根据系统负载动态调整

## 5. 跨平台支持方案

### 5.1 路径处理
- 使用`pathlib`进行跨平台路径操作
- 统一路径分隔符处理
- 支持Unicode文件名

### 5.2 系统集成
- **Windows**：右键菜单、发送到菜单、文件关联

### 5.3 打包部署
- **打包工具**：PyInstaller
- **Windows**：exe安装包 + 绿色版
- **macOS**：dmg镜像 + app bundle
- **Linux**：deb/rpm包 + AppImage

## 6. 性能优化策略

### 6.1 内存优化
1. **流式解压**：大文件分块处理，避免全量加载
2. **缓冲区管理**：自适应缓冲区大小（4KB-1MB）
3. **内存池**：重复使用内存对象，减少分配开销

### 6.2 CPU优化
1. **多核利用**：多文件并行解压
2. **算法选择**：根据文件类型选择最优解压算法
3. **预处理**：文件格式预检测，避免重复检测

### 6.3 IO优化
1. **异步文件操作**：减少UI线程阻塞
2. **批量写入**：减少磁盘寻址次数
3. **缓存策略**：常用目录缓存加速

## 7. 安全设计

## 10. 技术风险评估

### 10.1 风险点
1. **RAR格式支持**：需要unrar库，有版权限制
2. **性能瓶颈**：Python解释器性能限制
3. **打包体积**：PyInstaller打包后体积较大
4. **依赖管理**：多平台依赖库兼容性

### 10.2 缓解措施
1. **RAR替代方案**：使用系统unrar命令或提示用户安装
2. **性能优化**：关键部分使用Cython优化
3. **打包优化**：使用UPX压缩，移除未使用模块
4. **依赖管理**：使用虚拟环境，明确依赖版本
