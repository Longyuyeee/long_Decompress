## 测试标准和规范

### 1. 单元测试标准
- **AAA模式**: Arrange-Act-Assert
- **描述性测试名称**: 清晰描述预期行为
- **独立性**: 测试间无依赖
- **覆盖率目标**: 70%+ 代码覆盖率

### 2. 集成测试标准
- **真实数据流**: 测试模块间真实交互
- **错误处理**: 验证错误恢复机制
- **边界条件**: 测试各种边界情况

### 3. E2E测试标准
- **完整用户流程**: 测试端到端用户体验
- **跨平台兼容性**: 多浏览器测试
- **可访问性**: 支持键盘导航和屏幕阅读器

### 4. 性能测试标准
| 操作类型 | 文件大小 | 最大时间 | 最大内存 |
|---------|---------|---------|---------|
| 压缩 | 10MB | 3秒 | 50MB |
| 解压 | 10MB | 2秒 | 30MB |
| 批量处理 | 100MB | 15秒 | 200MB |

## 测试执行流程

### 本地开发测试
```bash
# 运行所有测试
npm test

# 运行单元测试
npm run test:unit

# 运行单元测试（监视模式）
npm run test:unit:watch

# 运行单元测试并生成覆盖率报告
npm run test:unit:coverage

# 运行E2E测试
npm run test:e2e

# 运行E2E测试UI
npm run test:e2e:ui

# Rust后端测试
cd src-tauri && cargo test

# Rust性能基准测试
cd src-tauri && cargo bench
```

### CI/CD测试流程
1. **代码推送/PR创建** → 触发GitHub Actions
2. **前端测试** → 单元测试 + 集成测试 + 覆盖率
3. **Rust测试** → 单元测试 + Clippy + 格式化检查
4. **E2E测试** → Playwright多浏览器测试
5. **性能测试** → 基准测试 + 性能监控
6. **安全扫描** → npm audit + cargo audit
7. **代码质量** → ESLint + Prettier + TypeScript检查
8. **质量门禁** → 所有测试通过才能合并
