## 数据库表设计

### 1. 主表结构

#### 1.1 密码条目表 (`password_entries`)
存储加密的密码条目信息。

```sql
CREATE TABLE password_entries (
    -- 基础信息
    id TEXT PRIMARY KEY,                    -- UUID v4
    name TEXT NOT NULL,                     -- 条目名称（可加密）
    username TEXT,                          -- 用户名（可加密）
    password TEXT NOT NULL,                 -- 密码（加密存储）
    url TEXT,                               -- 网址（可加密）
    notes TEXT,                             -- 备注（可加密）

    -- 分类和标签
    category TEXT NOT NULL,                 -- 分类：Personal, Work, Finance等
    tags TEXT NOT NULL DEFAULT '[]',        -- JSON数组：["tag1", "tag2"]

    -- 安全信息
    strength TEXT NOT NULL,                 -- 密码强度：VeryWeak, Weak, Medium, Strong, VeryStrong
    key_id TEXT NOT NULL,                   -- 加密密钥ID（外键到password_keys）
    encryption_algorithm TEXT NOT NULL,     -- 加密算法：AES256GCM
    encryption_version INTEGER NOT NULL DEFAULT 1, -- 加密版本

    -- 时间信息
    created_at DATETIME NOT NULL,
    updated_at DATETIME NOT NULL,
    last_used DATETIME,                     -- 最后使用时间
    expires_at DATETIME,                    -- 过期时间

    -- 状态标志
    favorite BOOLEAN NOT NULL DEFAULT FALSE,
    archived BOOLEAN NOT NULL DEFAULT FALSE, -- 是否归档
    deleted BOOLEAN NOT NULL DEFAULT FALSE,  -- 软删除标志

    -- 自定义字段
    custom_fields TEXT NOT NULL DEFAULT '[]', -- JSON数组：自定义字段

    -- 索引
    INDEX idx_password_entries_category (category),
    INDEX idx_password_entries_updated (updated_at),
    INDEX idx_password_entries_favorite (favorite),
    INDEX idx_password_entries_deleted (deleted),
    FOREIGN KEY (key_id) REFERENCES password_keys(id)
);
```

#### 1.2 密码组表 (`password_groups`)
管理密码分组。

```sql
CREATE TABLE password_groups (
    id TEXT PRIMARY KEY,                    -- UUID v4
    name TEXT NOT NULL,                     -- 组名称
    description TEXT,                       -- 组描述
    category TEXT NOT NULL,                 -- 组分类
    icon TEXT,                              -- 组图标
    color TEXT,                             -- 组颜色

    -- 安全设置
    require_master_password BOOLEAN NOT NULL DEFAULT TRUE, -- 是否需要主密码
    auto_lock_minutes INTEGER DEFAULT 30,   -- 自动锁定时间（分钟）

    -- 时间信息
    created_at DATETIME NOT NULL,
    updated_at DATETIME NOT NULL,

    -- 状态标志
    favorite BOOLEAN NOT NULL DEFAULT FALSE,
    archived BOOLEAN NOT NULL DEFAULT FALSE,

    -- 索引
    INDEX idx_password_groups_category (category),
    INDEX idx_password_groups_updated (updated_at)
);
```

#### 1.3 密码组关系表 (`password_group_entries`)
管理密码条目和组的关联关系（多对多）。

```sql
CREATE TABLE password_group_entries (
    id TEXT PRIMARY KEY,                    -- UUID v4
    group_id TEXT NOT NULL,                 -- 组ID
    entry_id TEXT NOT NULL,                 -- 条目ID
    added_at DATETIME NOT NULL,             -- 添加时间
    added_by TEXT,                          -- 添加者

    -- 索引
    UNIQUE (group_id, entry_id),            -- 防止重复添加
    INDEX idx_password_group_entries_group (group_id),
    INDEX idx_password_group_entries_entry (entry_id),
    FOREIGN KEY (group_id) REFERENCES password_groups(id) ON DELETE CASCADE,
    FOREIGN KEY (entry_id) REFERENCES password_entries(id) ON DELETE CASCADE
);
```

#### 1.4 加密密钥表 (`password_keys`)
管理加密密钥，支持密钥轮换。

```sql
CREATE TABLE password_keys (
    id TEXT PRIMARY KEY,                    -- UUID v4
    key_type TEXT NOT NULL,                 -- 密钥类型：Master, Entry, Group
    algorithm TEXT NOT NULL,                -- 算法：AES256GCM
    key_data TEXT NOT NULL,                 -- 加密的密钥数据（使用主密钥加密）
    key_hash TEXT NOT NULL,                 -- 密钥哈希（用于验证）

    -- 密钥信息
    key_size INTEGER NOT NULL,              -- 密钥大小（位）
    key_version INTEGER NOT NULL DEFAULT 1, -- 密钥版本
    derived_from TEXT,                      -- 派生自哪个密钥

    -- 使用限制
    max_usage_count INTEGER,                -- 最大使用次数
    usage_count INTEGER NOT NULL DEFAULT 0, -- 当前使用次数

    -- 时间信息
    created_at DATETIME NOT NULL,
    last_used_at DATETIME,                  -- 最后使用时间
    expires_at DATETIME,                    -- 过期时间
    rotated_at DATETIME,                    -- 轮换时间
    rotated_to TEXT,                        -- 轮换到哪个密钥

    -- 状态标志
    active BOOLEAN NOT NULL DEFAULT TRUE,   -- 是否激活
    archived BOOLEAN NOT NULL DEFAULT FALSE,

    -- 元数据
    metadata TEXT NOT NULL DEFAULT '{}',    -- JSON对象：元数据

    -- 索引
    INDEX idx_password_keys_type (key_type),
    INDEX idx_password_keys_active (active),
    INDEX idx_password_keys_expires (expires_at)
);
```

### 2. 审计和安全表

#### 2.1 密码审计表 (`password_audits`)
记录密码安全审计结果。

```sql
CREATE TABLE password_audits (
    id TEXT PRIMARY KEY,                    -- UUID v4
    entry_id TEXT NOT NULL,                 -- 条目ID
    audit_type TEXT NOT NULL,               -- 审计类型：Strength, Reuse, Breach, Expiry
    severity TEXT NOT NULL,                 -- 严重程度：Low, Medium, High, Critical
    score INTEGER NOT NULL,                 -- 安全评分（0-100）
    issues TEXT NOT NULL DEFAULT '[]',      -- JSON数组：问题列表
    recommendations TEXT NOT NULL DEFAULT '[]', -- JSON数组：建议列表

    -- 审计信息
    auditor TEXT,                           -- 审计者（系统或用户）
    audit_date DATETIME NOT NULL,           -- 审计日期
    next_audit_date DATETIME,               -- 下次审计日期

    -- 索引
    INDEX idx_password_audits_entry (entry_id),
    INDEX idx_password_audits_date (audit_date),
    INDEX idx_password_audits_severity (severity),
    FOREIGN KEY (entry_id) REFERENCES password_entries(id)
);
```

#### 2.2 密码使用历史表 (`password_usage_history`)
记录密码使用历史，用于安全分析和行为监控。

```sql
CREATE TABLE password_usage_history (
    id TEXT PRIMARY KEY,                    -- UUID v4
    entry_id TEXT NOT NULL,                 -- 条目ID
    user_id TEXT,                           -- 用户ID（如果有）
    action TEXT NOT NULL,                   -- 操作：View, Copy, AutoFill, Edit
    ip_address TEXT,                        -- IP地址
    user_agent TEXT,                        -- 用户代理
    device_info TEXT,                       -- 设备信息

    -- 时间信息
    used_at DATETIME NOT NULL,

    -- 索引
    INDEX idx_password_usage_entry (entry_id),
    INDEX idx_password_usage_date (used_at),
    INDEX idx_password_usage_action (action),
    FOREIGN KEY (entry_id) REFERENCES password_entries(id)
);
```

#### 2.3 数据泄露监控表 (`password_breach_monitor`)
监控密码是否在已知数据泄露中出现。

```sql
CREATE TABLE password_breach_monitor (

    entry_id TEXT NOT NULL,                 -- 条目ID
    breach_source TEXT NOT NULL,            -- 泄露来源
    breach_date DATETIME,                   -- 泄露日期
    exposed_data TEXT NOT NULL DEFAULT '[]', -- JSON数组：泄露的数据类型

    -- 监控状态
    detected_at DATETIME NOT NULL,          -- 检测时间
    notified_at DATETIME,                   -- 通知时间
    resolved_at DATETIME,                   -- 解决时间
    resolution TEXT,                        -- 解决方案

    -- 索引
    INDEX idx_password_breach_entry (entry_id),
    INDEX idx_password_breach_detected (detected_at),
    FOREIGN KEY (entry_id) REFERENCES password_entries(id)

```

### 3. 配置和元数据表

#### 3.1 密码策略表 (`password_policies`)

```sql
CREATE TABLE password_policies (

    prevent_reuse_count INTEGER DEFAULT 5,  -- 防止重用次数

    apply_to_categories TEXT DEFAULT '[]',  -- JSON数组：应用到的分类
    apply_to_groups TEXT DEFAULT '[]',      -- JSON数组：应用到的组

    enabled BOOLEAN NOT NULL DEFAULT TRUE,  -- 是否启用

    updated_at DATETIME NOT NULL,

    INDEX idx_password_policies_enabled (enabled),
    INDEX idx_password_policies_type (policy_type)
```


## 加密存储方案

### 1. 加密层次结构

```
主密码 (用户输入)
    ↓ (Argon2id哈希验证)
主密钥哈希 (存储验证)
    ↓ (Argon2id密钥派生)
主加密密钥
    ↓ (AES-256-GCM加密)
数据加密密钥 (每个条目独立)
    ↓ (AES-256-GCM加密)
密码数据 (加密存储)
```

### 2. 数据加密流程

#### 2.1 密码条目加密
```rust
// 1. 生成条目专用密钥
let entry_key = generate_aes256_key();

// 2. 加密敏感数据
let encrypted_password = aes256gcm_encrypt(
    plaintext_password,
    entry_key
);

let encrypted_username = aes256gcm_encrypt(
    plaintext_username,
    entry_key
);

// 3. 使用主密钥加密条目密钥
let encrypted_entry_key = aes256gcm_encrypt(
    entry_key,
    master_key
);

// 4. 存储加密数据
PasswordEntry {
    id: "uuid",
    password: encrypted_password,
    username: encrypted_username,
    key_id: encrypted_entry_key_id, // 关联到password_keys表
    // ... 其他字段
}
```

#### 2.2 密钥管理
- 每个密码条目使用独立密钥
- 密钥本身使用主密钥加密存储
- 支持密钥轮换和版本控制
- 密钥使用次数限制和过期机制

### 3. 数据库字段加密策略

| 字段名 | 是否加密 | 加密级别 | 说明 |
|--------|----------|----------|------|
| password | ✅ | 强加密 | AES-256-GCM |
| username | ✅ | 强加密 | AES-256-GCM |
| url | ⚠️ | 可选加密 | 根据敏感程度决定 |
| notes | ⚠️ | 可选加密 | 根据敏感程度决定 |
| custom_fields.value | ✅ | 条件加密 | 根据field_type决定 |
| name | ❌ | 不加密 | 用于搜索和显示 |
| category | ❌ | 不加密 | 分类信息 |
| tags | ❌ | 不加密 | 标签信息 |
