# 密码本数据库加密存储方案设计文档

## 概述
本文档描述了胧压缩·方便助手项目中密码本功能的加密存储方案。基于现有的crypto模块，设计了一个安全、可靠的密码本管理系统。

## 设计目标
1. **安全性**: 所有敏感数据（密码、自定义字段）必须加密存储
2. **可用性**: 提供完整的CRUD操作和搜索功能
3. **可扩展性**: 支持密码组、导入导出、审计等功能
4. **兼容性**: 与现有数据库模块和Tauri框架集成

## 架构设计

### 1. 核心组件

#### 1.1 加密密码服务 (`EncryptedPasswordService`)
- **职责**: 管理密码条目的加密存储和检索
- **特性**:
  - 主密码保护（使用Argon2id哈希）
  - AES-256-GCM加密所有敏感数据
  - 密钥管理集成（使用现有的`KeyManager`）
  - 文件系统存储（可扩展为数据库存储）

#### 1.2 密码组服务 (`PasswordGroupService`)
- **职责**: 管理密码分组和条目关联
- **特性**:
  - 创建、更新、删除密码组
  - 向组中添加/移除密码条目
  - 获取组中的所有条目

#### 1.3 Tauri命令接口 (`encrypted_password.rs`)
- **职责**: 提供前端可调用的安全API
- **特性**:
  - 完整的CRUD操作命令
  - 密码生成和审计功能
  - 导入导出支持（加密格式）

### 2. 数据流

```
前端 (Vue) → Tauri命令 → 加密密码服务 → 密钥管理器 → 加密存储
                    ↓
                密码组服务 → 组关系存储
```

### 3. 加密方案

#### 3.1 主密码保护
- 使用Argon2id算法哈希主密码
- 哈希值存储在单独的文件中
- 每次解锁需要验证主密码

#### 3.2 数据加密
- 每个密码条目使用独立的AES-256-GCM密钥加密
- 密钥本身使用主密码派生的密钥加密
- 自定义字段中的敏感数据单独加密

#### 3.3 密钥管理
- 使用现有的`KeyManager`管理加密密钥
- 支持密钥轮换和备份
- 密钥存储在加密的密钥库中

### 4. 存储结构

#### 4.1 文件系统布局
```
data/
├── master_password.hash      # 主密码哈希
├── password_vault.dat        # 加密的密钥库
├── passwords/               # 密码条目存储
│   ├── {entry_id}.json      # 加密的密码条目
│   └── ...
└── groups/                  # 密码组存储
    ├── {group_id}.json      # 密码组定义
    └── ...
```

#### 4.2 数据库集成（可选）
当前使用文件系统存储，但设计支持迁移到SQLite数据库：
- `password_entries`表存储加密的密码数据
- `password_groups`表存储组关系
- 使用现有的数据库迁移系统

### 5. API设计

#### 5.1 服务管理
- `init_encrypted_password_service()`: 初始化服务（设置主密码）
- `unlock_encrypted_password_service()`: 解锁服务
- `lock_encrypted_password_service()`: 锁定服务
- `is_encrypted_password_service_unlocked()`: 检查解锁状态

#### 5.2 密码条目管理
- `add_encrypted_password()`: 添加密码条目
- `get_encrypted_password()`: 获取密码条目
- `update_encrypted_password()`: 更新密码条目
- `delete_encrypted_password()`: 删除密码条目
- `search_encrypted_passwords()`: 搜索密码条目
- `list_encrypted_passwords()`: 列出所有密码条目

#### 5.3 高级功能
- `generate_strong_password()`: 生成强密码
- `audit_encrypted_passwords()`: 审计密码安全性
- `export_encrypted_passwords()`: 导出密码本（加密）
- `import_encrypted_passwords()`: 导入密码本

#### 5.4 密码组管理
- `create_password_group()`: 创建密码组
- `get_password_group()`: 获取密码组
- `update_password_group()`: 更新密码组
- `delete_password_group()`: 删除密码组
- `list_password_groups()`: 列出所有密码组
- `add_entry_to_password_group()`: 向组中添加条目
- `remove_entry_from_password_group()`: 从组中移除条目
- `get_group_entries()`: 获取组中的所有条目

### 6. 安全考虑

#### 6.1 内存安全
- 使用`zeroize`库安全清空内存中的敏感数据
- 密码在内存中解密后尽快使用和清除
- 使用安全缓冲区管理密钥数据

#### 6.2 加密强度
- AES-256-GCM提供认证加密
- Argon2id抵抗GPU和ASIC攻击
- 随机盐值和nonce防止重放攻击

#### 6.3 密钥管理
- 主密码不直接用于加密数据
- 使用密钥派生函数生成加密密钥
- 支持密钥轮换和备份

## 使用示例

### 初始化服务
```rust
let mut service = EncryptedPasswordService::new(data_dir);
service.initialize("master_password_123!").await?;
```

### 添加密码条目
```rust
let entry = PasswordEntry::new(
    "示例网站".to_string(),
    Some("user@example.com".to_string()),
    "StrongPassword123!".to_string(),
    Some("https://example.com".to_string()),
    PasswordCategory::Personal,
);

let encrypted_entry = service.add_password(entry).await?;
```

### 搜索密码
```rust
let results = service.search_passwords("示例").await?;
for entry in results {
    println!("找到: {}", entry.name);
}
```

### 导出密码本
```rust
let options = PasswordImportExportOptions {
    format: ImportExportFormat::Json,
    include_passwords: true,
    include_metadata: true,
    encrypt: true,
    password: Some("export_password".to_string()),
};

let export_data = service.export_passwords(&options, "export_password").await?;
```
