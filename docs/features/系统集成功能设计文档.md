# 胧压缩·方便助手 - 系统集成功能设计文档

## 1. 概述
系统集成功能负责将应用深度集成到操作系统中，提供更好的用户体验和系统级功能。包括系统通知、托盘图标、文件关联、右键菜单等。

## 2. 设计目标
1. **系统通知**: 任务完成、错误提示等系统通知
2. **托盘集成**: 系统托盘图标和菜单
3. **文件关联**: 关联压缩文件格式，支持双击打开
4. **右键菜单**: 文件管理器右键菜单集成
5. **系统服务**: 后台服务和开机启动
6. **跨平台支持**: Windows、macOS、Linux兼容

## 5. 跨平台实现

### 5.1 Windows实现
- **通知**: Windows Toast通知 API
- **托盘**: Shell_NotifyIcon API
- **文件关联**: 注册表操作
- **右键菜单**: 注册表上下文菜单
- **服务**: Windows服务 API

### 5.2 macOS实现
- **通知**: NSUserNotificationCenter
- **托盘**: NSStatusBar
- **文件关联**: Launch Services
- **右键菜单**: 服务菜单
- **服务**: Launch Agents

### 5.3 Linux实现
- **通知**: libnotify / DBus
- **托盘**: StatusNotifierItem / X11
- **文件关联**: MIME类型数据库
- **右键菜单**: 文件管理器扩展
- **服务**: systemd用户服务

## 6. 集成设计

### 6.1 与任务队列系统集成
```rust
// 任务完成时发送通知
task_manager.on_task_completed(|task| {
    if task.success {
        notifier.send_task_completed_notification(&task);
    } else {
        notifier.send_task_failed_notification(&task);
    }
});

// 任务进度更新时更新托盘图标
task_manager.on_task_progress(|task| {
    tray_manager.update_progress_badge(task.progress);
});
```

## 8. 用户体验设计

### 8.1 通知体验
- **非侵入式**: 通知不打断用户工作流
- **可操作**: 通知包含操作按钮（查看详情、重试等）
- **可配置**: 用户可配置通知类型和优先级
- **历史记录**: 保存通知历史，支持查看

### 8.2 托盘体验
- **状态指示**: 图标反映应用状态（空闲、忙碌、错误）
- **快速访问**: 常用功能一键访问
- **最小化到托盘**: 窗口最小化时隐藏到托盘
- **角标提醒**: 未读通知数量提醒

### 8.3 文件操作体验
- **无缝集成**: 文件操作与系统文件管理器无缝集成
- **智能识别**: 自动识别压缩文件格式
- **快速操作**: 右键菜单提供常用操作
- **批量处理**: 支持多文件批量操作
